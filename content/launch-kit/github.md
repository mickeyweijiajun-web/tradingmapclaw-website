## 4. v2.0 Release Note

**TradingMapClaw v2.0 — 2026-07-12**

This release marks the system's current stable state after roughly two months of solo development since May 2026.

**Architecture**
- Dual-engine research pipeline: Engine 1 (Hermes Agent) orchestrates fundamentals, valuation, insider activity, and macro/industry/sentiment passes; Engine 2 (Codex) independently runs technicals, capital flow, and options, and performs mandatory cross-verification of Engine 1's numbers.
- Multi-Model Council: DeepSeek V4 Pro, GLM-5.2, GPT-5.6, with a local Qwen3 14B closing the fallback chain. Council War Room tiebreaker order: DeepSeek → GLM-5.2 → GPT-5.6.
- Three weighted analysis passes per research cycle: Pass A fundamentals (40%, Engine 1), Pass B technicals + cross-verification (35%, Engine 2), Pass C macro/sentiment (25%, Engine 1).

**Scale (as of 2026-07-12)**
- 118 scheduled workflows (117 enabled)
- 230+ Python scripts
- 50+ skills
- 82 tracked assets across 5 groups
- 13 report types
- 12+ data sources
- Infrastructure: one Mac mini, ~$55/month budget cap, actual spend ~$7/month

**Compliance note**
All output remains research and education only. Scenario labels (bullish/neutral/bearish) exclusively — no buy/sell calls, no signals, no picks, no performance guarantees. WATCHLIST_ONLY: no brokerage integration, no trade execution, no auto-trading, no copy-trading.

**Known documentation note**
Some Tutorial PDFs from the early July edition still reference a "three-engine" description internally. This is a legacy phrasing issue, disclosed in the version note of those documents; the correct architecture is two engines (Hermes Agent, Codex) running three analysis passes. The next tutorial edition will unify this language.

Full write-up and system pages: https://www.tradingmapclaw.com
